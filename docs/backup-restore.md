# Backup and Restore

This repository uses FOSS backup tooling. The active backup path is Restic over
SSH from `shade` to `dusk`. Off-site Restic-over-SSH jobs are wired
declaratively but disabled until a third host exists.

## Current Topology

- `shade -> dusk`: active Restic repository at
  `/persist/backups/restic/shade` on `dusk`.
- `dusk -> local service dumps`: Forgejo and Vaultwarden write service backups
  under `/persist/backups`.
- `shade -> offsite`: declared but disabled.
- `dusk -> offsite`: declared but disabled.

The setup is not yet full 3-2-1. Do not mark that TODO complete until the
off-site SSH target exists, backups run there, and a restore drill passes.

## Active Shade Backup

As of 2026-07-16, the repository has a completed small restore-drill snapshot
tagged `codex-restore-drill`. The previous scheduled full-home runs had failed
while using an SSH jump-host SFTP path. The declarative configuration now uses
direct WireGuard SSH to `10.200.200.2`; after deploying `shade`, run the full
unit manually before treating the daily laptop backup as healthy.

Run a backup manually:

```sh
systemctl start restic-backups-shade-to-dusk.service
journalctl -u restic-backups-shade-to-dusk.service -n 100 --no-pager
```

List snapshots:

```sh
RESTIC_PASSWORD_FILE=/run/agenix/restic-shade-password \
RESTIC_REPOSITORY=sftp:usu@10.200.200.2:/persist/backups/restic/shade \
restic snapshots
```

The SFTP command pins Dusk's SSH host key through
`/etc/ssh/known_hosts.d/dusk-restic`; update that file only after verifying a
real Dusk host-key rotation.

## Restore Drill

Restore into a temporary directory on `shade`:

```sh
tmp="$(mktemp -d)"
restic restore latest \
  --tag codex-restore-drill \
  --repo sftp:usu@10.200.200.2:/persist/backups/restic/shade \
  --password-file /run/agenix/restic-shade-password \
  --target "$tmp" \
  --include /persist/home/usu/flake/TODO.md
find "$tmp" -maxdepth 4 -type f | sort | head
rm -rf "$tmp"
```

The drill passes when representative files are present and readable. Record the
date in a private note or issue before ticking restore-related TODOs.

For a full backup drill, deploy the direct-WireGuard Restic unit on `shade`,
run `systemctl start restic-backups-shade-to-dusk.service`, confirm it exits
successfully, and restore a non-trivial directory to a temporary path.

## Service State

Forgejo dumps are generated by the NixOS Forgejo module:

```sh
ssh dusk 'systemctl status forgejo-dump.service --no-pager'
ssh dusk 'find /persist/backups/forgejo -maxdepth 1 -type f -printf "%TY-%Tm-%Td %TH:%TM %p\n" | sort | tail'
```

Vaultwarden backups are generated by the NixOS Vaultwarden module:

```sh
ssh dusk 'systemctl status backup-vaultwarden.service --no-pager'
ssh dusk 'find /persist/backups/vaultwarden -maxdepth 1 -type f -printf "%TY-%Tm-%Td %TH:%TM %p\n" | sort | tail'
```

## Future Off-site Target

When a third host exists, enable `local.backups.offsiteRestic` on `shade` and
`dusk`, set:

- `target.host`
- `target.user`
- `target.repositoryPath`
- `target.hostKey`
- `identityFile`
- `passwordFile`

Then run each unit manually, list snapshots, and perform the restore drill from
the off-site repository before marking off-site backup work complete.
